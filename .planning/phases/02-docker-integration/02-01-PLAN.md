<objective>
Mount the host D-Bus socket into the Docker container and install the dbus-next npm package so the Nuxt server API routes can communicate with systemd.

**Purpose:** This phase bridges the Docker container and the host systemd. Without the D-Bus socket mount, the API routes (Phase 3) cannot communicate with the mcserver service. Without dbus-next installed, there is no D-Bus client library in the container.

**Output:**
- `package.json` with `dbus-next@0.10.2` added
- `docker-compose.yml` with D-Bus socket volume mount
- Container rebuilt and running with D-Bus access
- Verified: D-Bus connection test succeeds from inside the container
</objective>

2. **Test D-Bus connectivity from inside the container:**
   ```bash
   docker exec hjem node -e "
   const { MessageBus, DBusError } = require('dbus-next');
   async function test() {
     try {
       const bus = new MessageBus();
       await bus.connect({ bus: 'system' });
       console.log('D-Bus connection: OK');
       const obj = await bus.getProxyObject('org.freedesktop.systemd1', '/org/freedesktop/systemd1');
       const iface = obj.getInterface('org.freedesktop.systemd1.Manager');
       const state = await iface.GetUnitFileState('mcserver.service');
       console.log('systemd reachable: OK (mcserver state:', state, ')');
       await bus.disconnect();
       process.exit(0);
     } catch (e) {
       console.error('FAIL:', e.message);
       process.exit(1);
     }
   }
   test();
   "
   ```
   Expected: Both "D-Bus connection: OK" and "systemd reachable: OK" print without errors.

   Expected: shows `hjem` with `healthy` or `Up` status.
