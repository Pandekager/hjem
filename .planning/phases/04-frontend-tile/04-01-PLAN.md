<interfaces>
**API endpoints (from Phase 3):**
- `GET /api/mc/status` → `{ state: "active" | "inactive" | "activating" | "deactivating" | "failed" | "not-found" }`
- `POST /api/mc/start` → `{ success: true }` or 409
- `POST /api/mc/stop` → `{ success: true }` or 409

**State mapping:**
- `"active"` → server running (green dot)
- `"inactive"`, `"not-found"` → server stopped (gray dot)
- `"activating"` → transitional (yellow dot)
- `"deactivating"` → transitional (orange dot)
- `"failed"` → server crashed (red dot)
- API call error → error state (red dot + error message)

**Visual consistency with AppTile:**
- Same dimensions: min-height 260px, border-radius 1.5rem, padding 1.4rem
- Same overlay gradient, glow effect
- Same tile class structure, font sizes, spacing
- Same hover effect (translateY(-6px) rotate(-0.4deg))
- Same tile-content, tile-heading, tile-icon, tile-name classes

**Edge cases:**
- Race condition: rapidly clicking start then stop — frontend disables buttons immediately
- `not-found` means stopped (systemd unloaded the unit)
- API 409 means action was rejected — show brief error feedback
- Network error → error state with retry button
</interfaces>
